画图题指南

1. 用例图 (Use Case Diagram)

组成元素：

  • 参与者 (Actor)：与系统交互的外部实体，用小人图标表示
  • 用例 (Use Case)：系统提供的功能，用椭圆表示
  • 系统边界：矩形框，区分系统内外
  • 关系：
    • 关联关系：参与者与用例之间的连线
    • 包含关系 (<>)：必须执行的基础功能
    • 扩展关系 (<>)：可选的扩展功能
    • 泛化关系：继承关系，用带空心箭头的线表示

绘制步骤：

  • 1. 识别系统的参与者（用户、其他系统）
  • 2. 识别系统的功能（用例）
  • 3. 确定系统边界
  • 4. 绘制参与者与用例的关联关系
  • 5. 识别用例之间的包含、扩展关系

示例（酒店预订系统）：

参与者：用户、管理员
用例：登录、搜索酒店、预订酒店、取消预订、管理酒店信息
关系：搜索酒店 <> 登录；取消预订 <> 预订酒店

2. 顺序图 (Sequence Diagram)

组成元素：

  • 对象/类：用矩形表示，顶部写类名
  • 生命线：从对象向下延伸的虚线
  • 激活期：生命线上的窄矩形，表示对象活跃时间
  • 消息：对象之间的水平箭头
    • 同步消息：实心箭头
    • 异步消息：空心箭头
    • 返回消息：虚线箭头
  • 片段：如 alt（条件）、loop（循环）、opt（可选）

绘制步骤：

  • 1. 确定参与交互的对象（边界类、控制类、实体类）
  • 2. 画生命线
  • 3. 按时间顺序绘制消息
  • 4. 添加激活期
  • 5. 添加条件和循环片段

三层架构：

  • 边界层：负责与用户交互（界面类）
  • 控制层：负责业务逻辑（控制类）
  • 实体层：负责数据存储（实体类）

3. 类图 (Class Diagram)

组成元素：

  • 类：矩形，分三层
    • 类名
    • 属性：可见性 + 名称 : 类型
    • 方法：可见性 + 名称(参数) : 返回类型
  • 可见性：+ public, - private, # protected
  • 关系：
    • 关联：实线，可带多重性（1, 0..1, *, 0..*）
    • 聚合：实线 + 空心菱形（整体与部分，可独立存在）
    • 组合：实线 + 实心菱形（整体与部分，不可独立）
    • 继承：虚线 + 空心箭头（泛化）
    • 实现：虚线 + 空心箭头（实现接口）
    • 依赖：虚线 + 箭头（使用关系）

绘制步骤：

  • 1. 识别系统中的类
  • 2. 定义类的属性和方法
  • 3. 确定类之间的关系
  • 4. 绘制类图
  • 5. 添加多重性和可见性

4. 状态图 (State Diagram)

组成元素：

  • 状态：圆角矩形，表示对象的条件
  • 转换：箭头，表示状态变化
  • 事件：触发转换的动作
  • 初始状态：实心圆
  • 终态：实心圆 + 外圈

绘制步骤：

  • 1. 识别对象的所有状态
  • 2. 确定状态之间的转换
  • 3. 识别触发转换的事件
  • 4. 绘制状态图

示例（订单状态）：

初始状态 → 待支付 → 已支付 → 待发货 → 已发货 → 已完成 → 终态
待支付 → 已取消（用户取消）
已支付 → 退款中 → 已退款

5. 活动图 (Activity Diagram)

组成元素：

  • 活动：圆角矩形，表示执行的工作
  • 转换：箭头，表示活动间的流转
  • 分支：菱形，表示条件判断
  • 合并：菱形，分支汇合
  • 分叉/汇合：粗线，表示并发执行
  • 初始节点：实心圆
  • 活动终节点：实心圆 + 外圈

绘制步骤：

  • 1. 确定活动的起始和结束
  • 2. 识别所有活动
  • 3. 确定活动间的顺序和条件
  • 4. 绘制活动图

示例（用户登录流程）：

初始 → 输入用户名密码 → 验证信息 → [信息正确] 登录成功 → 进入系统
[信息错误] 显示错误提示 → 重新输入 → 验证信息

测试用例设计

一、白盒测试 - 基本路径测试法

步骤：

  • 1. 绘制程序流图（节点表示语句或判定，边表示控制流）
  • 2. 计算圈复杂度：V(G) = 边数 - 节点数 + 2
  • 3. 确定基本路径集合（独立路径的数量等于圈复杂度）
  • 4. 为每条基本路径设计测试用例

示例：求最大公约数

int gcd(int x, int y) {
    if (x < y) {           // 判定1
        交换x和y;
    }
    while (x % y != 0) {   // 判定2
        int y1 = y;
        y = x % y;
        x = y1;
    }
    return y;
}

流图分析：

节点数 N = 6，边数 E = 7
圈复杂度 V(G) = 7 - 6 + 2 = 3
基本路径数量 = 3 条

测试用例：

用例编号 基本路径 x, y 值 预期输出
1 路径1：x>=y, x%y=0 x=6, y=3 3
2 路径2：x x=3, y=6 3
3 路径3：x>=y, x%y!=0, 循环后 x%y=0 x=12, y=8 4

二、黑盒测试 - 等价类划分法

步骤：

  • 1. 划分等价类（有效等价类、无效等价类）
  • 2. 为每个等价类编号
  • 3. 设计测试用例：
    • 用尽可能少的测试用例覆盖尽可能多的有效等价类
    • 为每个无效等价类设计1个测试用例

示例：密码验证

需求：密码由数字和字母组成，长度为6-12位

等价类划分：

输入条件 有效等价类 无效等价类
字符类型 (1) 包含数字和字母 (2) 只包含数字
(3) 只包含字母
(4) 包含特殊字符
长度 (5) 6-12位 (6) 小于6位
(7) 大于12位

测试用例：

用例编号 测试输入 预期结果 覆盖等价类
1 abc123 通过 (1)(5)
2 123456 拒绝 (2)
3 abcdef 拒绝 (3)
4 abc@12 拒绝 (4)
5 ab1 拒绝 (6)
6 abc123456789 拒绝 (7)

三、黑盒测试 - 边界值分析法

原则：

  • 错误更容易发生在输入边界附近
  • 选择边界值及其邻近值作为测试用例

示例：密码长度 6-12 位

测试值 说明
5 略小于最小值
6 最小值
7 略大于最小值
11 略小于最大值
12 最大值
13 略大于最大值

分析与设计题

例题1：酒店预订系统

需求描述：
用户在使用系统"酒店预订"功能之前必须登录。用户首先在酒店预订界面上选择预订信息，包括城市、入住日期、退房日期、房间数量、成人和儿童人数等；用户点击搜索按钮，系统进行检查，若城市、入住日期、退房日期为空或入住日期晚于退房日期，则在预订界面上提示出错，否则进行搜索并返回在新的搜索结果页面上以列表形式展示搜索结果，搜索结果若为空集则在搜索结果页面上告知用户没有符合要求的酒店，请调整搜索条件。要求搜索时间不超过10秒。
要求：
  • (1) 画出"用户"角色的用例图模型
  • (2) 分别画出"搜索酒店"和"登录"的顺序图模型（包括边界类、控制类和实体类）
  • (3) 画出分析类图模型
解题思路：
  • 用例图：
    • 参与者：用户
    • 用例：登录、搜索酒店、预订酒店
    • 关系：搜索酒店 <> 登录
  • 顺序图（搜索酒店）：
    • 边界类：搜索界面、搜索结果界面
    • 控制类：搜索控制器
    • 实体类：酒店信息
    • 流程：用户 → 搜索界面 → 搜索控制器 → 酒店信息 → 搜索控制器 → 搜索结果界面 → 用户
  • 类图：
    • 边界类：搜索界面（显示搜索表单、接收输入）、搜索结果界面（显示结果列表）
    • 控制类：搜索控制器（验证输入、执行搜索）
    • 实体类：酒店信息（酒店ID、名称、地址、价格）

例题2：在线购物车系统

需求描述：
用户在使用系统之前必须登录。用户可以浏览商品并将商品添加到购物车。用户可以查看购物车中的商品列表，并对购物车中的商品进行修改。用户可以提交购物车以生成订单。在订单生成之前，系统会对购物车中的商品库存进行检查，若库存不足则提示用户修改购物车。若所有商品库存充足，则生成订单并清空购物车。
要求：
  • (1) 画出"用户"角色的用例图模型
  • (2) 分别画出"浏览商品"和"提交购物车"的顺序图模型
  • (3) 画出分析类图模型
解题思路：
  • 用例图：
    • 参与者：用户
    • 用例：登录、浏览商品、添加到购物车、查看购物车、修改购物车、提交订单
    • 关系：浏览商品 <> 登录；提交订单 <> 查看购物车
  • 顺序图（提交购物车）：
    • 边界类：购物车界面、订单界面
    • 控制类：订单控制器
    • 实体类：购物车、商品、订单
    • 流程：用户 → 购物车界面 → 订单控制器 → 商品（检查库存）→ 订单控制器 → 订单（生成订单）→ 订单界面 → 用户
  • 类图：
    • 边界类：购物车界面、订单界面
    • 控制类：订单控制器
    • 实体类：购物车（购物车ID、商品列表）、商品（商品ID、名称、价格、库存）、订单（订单ID、商品列表、总价）

例题3：界面设计类图

题目：
以下界面是某款APP"绑定手机号"功能的设计界面。请给出该界面的设计类图。
解题思路：
  • 识别界面元素：
    • 输入框：手机号输入框、验证码输入框
    • 按钮：获取验证码按钮、确认绑定按钮
    • 提示文本
  • 设计类图：
    • 边界类：BindPhoneView（显示界面、接收输入）
    • 控制类：BindPhoneController（验证手机号、发送验证码、绑定手机号）
    • 实体类：User（用户ID、手机号）
  • 属性和方法：
    • BindPhoneView: -phoneNumber: String, -verifyCode: String, +display(), +getInput()
    • BindPhoneController: +validatePhone(), +sendVerifyCode(), +bindPhone()
    • User: -userId: int, -phone: String, +getPhone(), +setPhone()
